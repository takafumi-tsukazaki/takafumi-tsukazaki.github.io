Subdifferential Visualizer — ∂f(x₀)

The subdifferential ∂f(x₀) of a convex function is the set of slopes g of all affine functions that support the graph from below: f(z) ≥ f(x₀) + g·(z − x₀). At smooth points it is a single number {f′(x₀)}; at a kink it spreads into a whole interval [f′₋, f′₊]. Drag the point x₀ along the curve (left panel) and watch the set ∂f(x₀) (right panel) — where it crosses g = 0 is a minimizer (0 ∈ ∂f). The bottom panel shows how that same fact drives the proximal operator and sparsity.

Proximal optimality & soft-thresholding
0 ∈ ∂f(x) + (x − v)/λ ⟺ v − x ∈ λ·∂f(x)
f = |·| : proxλ|·|(v) = sign(v)·max(|v| − λ, 0)  (soft-threshold)
Because 0 ∈ ∂|·|(0) = [−1, 1] is a fat set, every v ∈ [−λ, λ] is pulled exactly to 0 — that dead-zone is sparsity (the LASSO mechanism). A smooth regularizer (0.45·x²) has 0 ∈ ∂f at a single point, so prox is linear shrinkage v / (1 + 0.9λ) and never reaches exact zero.
What does this panel show? (for the selected f, λ, v)

1 · What it computes

The proximal operator is the solution of a tug-of-war: proxλf(v) = argminₓ { f(x) + (x − v)² / (2λ) }. The penalty (x − v)²/(2λ) says "stay near v"; the term f(x) says "make f small". Their compromise is the output x. The curve plots this map for every input v (horizontal axis) to its output x = proxλf(v) (vertical axis), at the currently selected f and λ.

2 · What each input means

  • f(x) — the regularizer / penalty. The shape of its subdifferential sets the shape of the map: a kink (where 0 ∈ ∂f is a fat interval) creates a flat zero region (sparsity); a smooth f never does.
  • λ — the weight between data (v) and penalty (f). λ → 0 makes prox(v) → v (the curve hugs the identity line); λ → ∞ makes prox(v) → argmin f (everything is pulled to a minimizer). The dead-zone width grows with λ.
  • v — the input point being mapped. Read it as a noisy observation, or the current iterate in an optimization step.

3 · How to read the picture

  • Amber dead-zone — the inputs v whose output sticks exactly on a minimizer of f (0 ∈ ∂f(prox(v))). For f = |x| this is v ∈ [−λ, λ], all mapped to exactly 0: that is sparsity.
  • Dashed identity line y = v — the no-shrinkage baseline; the gap down to the curve is how far v is pulled.
  • Orange arrow — the shrinkage v − x for the current v.
  • Readout (v − x)/λ ∈ ∂f(x) — this is the optimality condition that characterizes the prox: 0 ∈ ∂f(x) + (x − v)/λ ⟺ (v − x)/λ ∈ ∂f(x). A ✓ means the displacement, scaled by 1/λ, is a valid subgradient at the output x.

Block / coordinate soft-thresholding
ℓ₁: proxλ‖·‖₁(v)ᵢ = sign(vᵢ)·max(|vᵢ| − λ, 0)  (per coordinate)
ℓ₂: proxλ‖·‖₂(v) = max(1 − λ/‖v‖, 0)·v  (whole vector)
ℓ₁ has independent dead-zones per axis, so a coordinate hits exactly 0 whenever |vᵢ| ≤ λ — that is why LASSO produces sparse solutions on the axes. ℓ₂'s subdifferential at 0 is a round disk, so it can only kill the entire vector at once (‖v‖ ≤ λ): no per-coordinate sparsity.
